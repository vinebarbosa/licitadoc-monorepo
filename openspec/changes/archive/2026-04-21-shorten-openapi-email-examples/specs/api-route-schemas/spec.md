## ADDED Requirements

### Requirement: Application-owned route contracts MUST expose readable email examples
The API MUST allow application-owned Zod-backed route schemas to declare explicit, concise examples for email fields so Swagger/OpenAPI does not fall back to oversized autogenerated values.

#### Scenario: Documenting an email field in a route response
- **WHEN** a route response schema in `apps/api` exposes an email field
- **THEN** the exported OpenAPI document includes a short, human-readable example for that field
- **AND** Swagger UI renders that readable example instead of a very large autogenerated string

#### Scenario: Documenting an email field in a route request body
- **WHEN** a route request schema in `apps/api` accepts an email field
- **THEN** the exported OpenAPI document includes a short, human-readable example for that field

### Requirement: Readable email examples MUST remain compatible with the generated OpenAPI contract
The API MUST preserve explicit email examples in the exported OpenAPI document without breaking the existing contract export and client-generation workflow.

#### Scenario: Regenerating the API contract after adding explicit email examples
- **WHEN** the API exports or serves `/openapi.json` after the email example metadata is added
- **THEN** the document contains the explicit email examples for the affected fields
- **AND** the existing client-generation workflow can still consume the exported document
